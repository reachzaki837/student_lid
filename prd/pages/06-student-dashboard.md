# Student Dashboard

> **Route:** `/dashboard` (when `user.role == "student"`)
> **Module:** Student
> **Generated:** 2026-04-10

## Overview

The Student Dashboard is the main hub for students after login. It shows their **learning persona** (dominant style combination from their most recent assessment), **personalized study recommendations** (tips, tools, schedule) generated by AI, a **VARK radar chart** visualizing their scores across all four VARK dimensions, and a **class join widget** to enroll in a teacher's class. Students who have not yet taken the assessment see a CTA prompt to do so.

## Fields

### Join Class Widget (Sidebar)

| Field | Type | Required | Default | Validation | Notes |
|-------|------|----------|---------|------------|-------|
| Class Code | Text input | Yes | — | Max 6 characters, auto-uppercased | 6-character alphanumeric code generated by a teacher for each class |

### No-Assessment State

No form fields — just a centered call-to-action card with a "Start Assessment" button linking to `/assessment/`.

## Interactions

### Page Load
- Authenticates user via JWT cookie; redirects to `/auth/login` if unauthenticated
- Loads the most recent `Assessment` document for the current user (sorted by `created_at` descending)
- Loads all `Enrollment` records for the current user to show joined classes
- Loads email profile from `ScoringService.get_student_email_profile()` (auto-parsed from email)
- Renders either the "has assessment" view or the "no assessment" empty state

### View Learning Persona (Has Assessment)
- Displays the `dominant_style` field (e.g., "The Visual Activist") in the persona banner
- Shows personalized study strategy card with three sections:
  - **Top Strategies:** List of tips (from AI or fallback static rules), each in a green-bordered item
  - **Best Tools:** List of recommended apps/tools (AI-generated or static fallback)
  - **Ideal Schedule:** Block of text describing the recommended study schedule
- Renders a **Chart.js radar chart** showing all four VARK scores (Visual, Aural, Read/Write, Kinesthetic) plotted on axes

### Join Class
- **Trigger:** User enters a class code (up to 6 uppercase alphanumeric chars) and clicks "Join"
- **API:** POST form to `/dashboard/join_class`
- **Success:** Redirects to `/dashboard` with updated class list
- **Invalid code:** Re-renders the student dashboard template with `error: "Invalid class code"` displayed at top
- **Already enrolled:** Silently redirects to `/dashboard` without error (existing enrollment is detected but not reported)
- **Already enrolled in a different class:** The student can join multiple classes (no limit)

### Empty State (No Assessment)
- Shows a clipboard icon, "Discover Your Learning Style" heading, and descriptive text
- "Start Assessment" button navigates to `/assessment/`
- No recommendations or chart shown

### Settings Navigation
- "Account Settings" sidebar link navigates to `/dashboard/settings`

### AI Study Tutor Navigation
- "AI Study Tutor" sidebar link navigates to `/dashboard/tutor`
- Labeled with a "PRO" badge

### Logout
- Clicking the logout icon in the user profile section navigates to `/auth/logout`, which clears the JWT cookie and redirects to `/auth/login`

## API Dependencies

| API | Method | Path | Trigger | Notes |
|-----|--------|------|---------|-------|
| Dashboard Load | GET | `/dashboard` | Page navigation | Renders student or teacher template based on role |
| Join Class | POST | `/dashboard/join_class` | Join form submit | Creates Enrollment document if code is valid |
| Assessment Submit | POST | `/assessment/submit` | Assessment form | Creates Assessment document, redirects to dashboard |

## Page Relationships

- **From:** Login, Register (post-registration), Assessment submit, AI Tutor
- **To:** `/assessment/` (Start Assessment / Take Assessment link), `/dashboard/tutor` (AI Study Tutor), `/dashboard/settings`, `/auth/logout`
